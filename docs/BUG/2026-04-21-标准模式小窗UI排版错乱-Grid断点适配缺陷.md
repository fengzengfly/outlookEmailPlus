- **受影响页面**：邮箱管理页（`#page-mailbox`）

## 根本原因分析

### 核心问题：Grid 四栏布局的最小宽度总和 > 中等窗口宽度

**定位：**
- `static/css/layout.css` 第 44–58 行（Grid 四栏布局定义）
- `static/css/layout.css` 第 10–42 行（CSS 变量：各面板最小/默认宽度）
- `static/js/layout-manager.js` 第 1042–1078 行（`handleWindowResize` 断点逻辑）
- `static/css/main.css` 第 1337–1352 行（平板断点 `@media` 规则）

**Grid 布局定义（layout.css:44-58）：**

```css
.main-container {
  display: grid;
  grid-template-columns:
    minmax(var(--group-panel-layout-min-width), var(--group-panel-layout-width))     /* 150~200px */
    minmax(var(--account-panel-layout-min-width), var(--account-panel-layout-width))   /* 180~260px */
    minmax(var(--email-list-panel-layout-min-width), var(--email-list-panel-layout-width)) /* 280~380px */
    minmax(var(--email-detail-panel-min-width), 1fr);                                /* 400px~1fr  */
  grid-template-rows: 1fr;
  overflow: hidden;    /* ← 内容被裁剪，不换行也不滚动 */
}
```

**四栏最小宽度合计：**

```
group min:  150px
account min: 180px
email min:   280px
detail min:  400px
─────────────────
合计最小：  1010px
```

**加上侧边栏（220px）和 topbar：**

```
侧边栏：220px
四栏最小：1010px
─────────────────
合计最小：1230px
```

**当窗口宽度 < 1230px 时，Grid 列被压缩到 minmax 下限以下，导致内容溢出、重叠。**

---

### 缺陷一：JS 断点与实际 Grid 需求不匹配

**定位：** `layout-manager.js` 第 1042–1078 行

```javascript
handleWindowResize() {
  const width = window.innerWidth;

  const shouldCollapseGroups = width < 1200;   // ← 仅按 innerWidth
  const shouldCollapseAccounts = width < 900;
  const shouldCollapseEmails = width < 700;
  // ...
}
```

**问题：**

1. **断点阈值未考虑侧边栏宽度**。`window.innerWidth` 是整个视口宽度，包含了侧边栏（220px）。当 `innerWidth = 1200px` 时，实际给 `.main-container` 的可用宽度仅为 `1200 - 220 = 980px`，远不足以容纳四栏（最小合计 1010px）。

2. **断点与 Grid minmax 下限不同步**。

   | 面板 | Grid min | 折叠断点（innerWidth） | 折叠时 Grid 可用宽度 |
   |------|----------|----------------------|---------------------|
   | groups | 150px | <1200 | `1200-220=980` → 剩 3 栏需 ≥860px（account 180+email 280+detail 400），实际 OK |
   | accounts | 180px | <900 | `900-220=680` → 剩 2 栏需 ≥680px（email 280+detail 400），刚好 |
   | emails | 280px | <700 | `700-220=480` → 剩 1 栏需 ≥400px，刚好 |

   看似刚好满足，但：
   - 侧边栏在 `≥1024px` 时为完整 220px，`769~1024px` 时缩为 60px（icon-only）
   - `handleWindowResize` 没有根据侧边栏实际宽度动态计算断点
   - **1024px~1200px 之间存在盲区**：侧边栏 220px + 三栏（groups 150+account 180+email 280 = 610px）+ detail(400) = 1230px，而窗口只有 1024~1200px

3. **没有处理 `.content` 容器的实际可用宽度**。在桌面端，`.content` 是 flex 子元素，`flex: 1`，实际宽度 = `window.innerWidth - sidebarWidth`。但 `handleWindowResize` 直接用 `innerWidth` 判断。

**实际效果：在 1024px~1200px 窗口范围内，四栏全部展开，但可用空间不足，Grid 被压缩。**

---

### 缺陷二：CSS `@media` 规则与 JS Grid 变量冲突

**定位：** `main.css` 第 1337–1352 行

```css
/* 平板断点 */
@media (max-width: 1024px) and (min-width: 769px) {
  .sidebar { width: 60px; overflow: hidden; }
  /* ... */
  .groups-column { width: 180px; }   /* ← 旧 flex 布局的固定宽度 */
  .accounts-column { width: 240px; }  /* ← 旧 flex 布局的固定宽度 */
}
```

**问题：**

1. 这些 `width: 180px` / `width: 240px` 是给旧 `.mailbox-layout`（flex 布局）写的，但新布局系统（`layout.css`）使用的是 CSS Grid + CSS 变量控制列宽。
2. Grid 子元素的 `width` 属性会被 Grid 布局覆盖（Grid track 尺寸优先于子元素 width），所以这些 CSS 规则**对新布局系统无效**。
3. `layout.css` 没有对应的 `@media` 规则来在平板断点下调小 CSS 变量值。

---

### 缺陷三：`.main-container` 的 `overflow: hidden` 隐藏了问题但未解决

**定位：** `layout.css` 第 54 行

```css
.main-container {
  overflow: hidden;  /* ← 防止折叠时面板内容溢出撑出滚动条 */
}
```

**问题：**

该属性是为折叠动画设计的（防止折叠面板的内容溢出导致出现全局滚动条），但副作用是：**当 Grid 被压缩到超出可用宽度时，超出部分被静默裁剪**，用户看不到被截断的内容，面板间出现重叠和错位的视觉效果。

---

### 缺陷四：简洁模式也有类似问题

**定位：** `main.css` 第 1406–1407 行

```css
@media (max-width: 768px) {
  .mail-head { display: none; }    /* 移动端隐藏表头 */
}
```

但简洁模式的 `.mail-row` 使用了 `grid-template-columns: 54px minmax(220px, 1.2fr) minmax(130px, 0.7fr) minmax(260px, 1.4fr) minmax(160px, 1fr) minmax(150px, 0.8fr)`，最小宽度合计约 974px。在 768px~974px 之间同样存在内容挤压问题。

## 按实际生产代码重新分析

### 实际根因：`.workspace` flex 三栏在平板断点无自适应

**当前生产布局（非 Grid）：**
- 模板：`templates/index.html` 中使用 `<div class="workspace workspace-mailbox">`
- CSS：`static/css/main.css` 中 `.workspace { display: flex; }`
- 三栏结构：
  - `.groups-column { width: 220px; flex-shrink: 0; }`
  - `.accounts-column { width: 280px; flex-shrink: 0; }`
  - `.emails-column { flex: 1; min-width: 0; }`

**平板断点（769px~1024px）实际现状：**
- `@media (max-width: 1024px) and (min-width: 769px)` 只处理了 sidebar（缩为 60px）
- `.workspace` 仍为 `flex-direction: row`
- `.groups-column { width: 180px; }` 和 `.accounts-column { width: 240px; }` 仍为固定宽度
- 三栏最小总宽度 ≈ 180 + 240 + 300(emails内容) = **720px+**
- 可用宽度 = 800 - 60(sidebar) = **740px**，emails 栏内容（搜索栏、+按钮、分页）开始挤压错位

**结论**：BUG 的**真实触发条件**与**真实根因**与 Grid 系统无关，而是 `.workspace` flex 三栏在平板断点缺少响应式处理。

## 原 Grid 布局分析（已废弃，仅供参考）

> 以下内容为原始分析，基于已废弃的 `layout.css` Grid 系统，不再适用于当前生产代码。

### 方案一（原推荐，已废弃）：JS 断点改用 `.main-container` 实际宽度

**文件：** `static/js/layout-manager.js`

**修改位置：** `handleWindowResize()` 方法

**核心思路：** 将断点判断基准从 `window.innerWidth` 改为 `.main-container` 的实际渲染宽度（即扣除侧边栏后的可用宽度）。

```javascript
handleWindowResize() {
  // 优先使用 .main-container 实际宽度（扣除侧边栏后的可用空间）
  const container = this.container;
  const availableWidth = container
    ? container.getBoundingClientRect().width || container.clientWidth
    : window.innerWidth;

  // 断点基于实际可用宽度（而非 window.innerWidth）
  const shouldCollapseGroups = availableWidth < 800;
  const shouldCollapseAccounts = availableWidth < 600;
  const shouldCollapseEmails = availableWidth < 400;
  // ...
}
```

**断点推导：**

| 面板 | 折叠后剩余面板最小宽度 | 建议断点 |
|------|----------------------|---------|
| groups 折叠 | account(180) + email(280) + detail(400) = 860px | `<860` |
| accounts 折叠 | email(280) + detail(400) = 680px | `<680` |
| emails 折叠 | detail(400) = 400px | `<400` |

预留一定余量后取整：
- groups: `<800`
- accounts: `<600`
- emails: `<400`

**优点：**
- 最小改动（只改断点值和判断基准）
- 自动适配侧边栏在不同断点下的实际宽度
- 保持现有折叠/恢复逻辑不变

---

### 方案二（配合方案一）：`layout.css` 增加中等断点的 CSS 变量覆盖

**文件：** `static/css/layout.css`

**新增内容：** 在文件末尾添加 `@media` 规则

```css
/* ===== 响应式：中等屏幕（降低面板最小宽度） ===== */
@media (max-width: 1100px) {

    --group-panel-width: 160px;
    --account-panel-min-width: 160px;
    --account-panel-width: 220px;
    --email-list-panel-min-width: 240px;
    --email-list-panel-width: 320px;
    --email-detail-panel-min-width: 300px;
  }
}

@media (max-width: 900px) {
  :root {
    --group-panel-min-width: 100px;
    --group-panel-width: 140px;
    --account-panel-min-width: 140px;
    --account-panel-width: 180px;
    --email-list-panel-min-width: 200px;
    --email-list-panel-width: 260px;

```

**优点：**
- 让 Grid 在中屏下有更合理的默认宽度
- 与 JS 自动折叠配合，提供"先缩窄、再折叠"的渐进体验

**文件：** `static/css/main.css`

**修改位置：** 第 1337–1352 行（平板断点）

```css
/* 删除或注释掉这些对 Grid 布局无效的旧规则 */
```
