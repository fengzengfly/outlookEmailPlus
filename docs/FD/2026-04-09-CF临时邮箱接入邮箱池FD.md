## 1. 功能定义

### 1.1 核心功能

将 Cloudflare Workers Temp Email (CF临时邮箱) 接入现有邮箱池系统,实现:

1. **动态创建**: 调用方通过 `/api/external/pool/claim-random`（body 传 provider）可动态创建 CF 临时邮箱
2. **统一调度**: CF邮箱与 Outlook/IMAP 邮箱统一管理,混合调度
3. **自动回收**: 使用完成后自动删除远程CF邮箱,释放资源
4. **无缝读取**: 邮件读取/验证码提取接口透明支持CF邮箱

### 1.2 功能边界

**本期包含**:
- `pool.claim_atomic()` 支持 CF 邮箱动态创建
- `pool.complete_claim()` 支持 CF 邮箱自动删除
- Schema v19 迁移 (accounts 表扩展)
- 前端账号列表显示 CF 邮箱 (只读)

**本期不包含**:
- 手动创建 CF 邮箱的 UI 界面
- `temp_emails` 表历史数据迁移
- CF 邮箱池容量上限控制
- 定期清理失败未删除的远程邮箱

### 2.2 数据流

**创建流程**:
```
External API(/claim-random) → pool.claim_atomic() 
            → temp_mail_provider_cf.create_mailbox() 
            → CF Worker API (POST /admin/new_address)
            → INSERT accounts (provider=cloudflare_temp_mail)
            → 返回 {email, claim_token}
```

**删除流程**:
```
External API(/claim-complete) → pool.complete_claim() 
            → UPDATE accounts (pool_status=used)
            → temp_mail_provider_cf.delete_mailbox()
            → CF Worker API (DELETE /admin/delete_address/{id})
            → 记录日志
```

### 3.2 temp_mail_meta 字段规范

**字段来源**: `temp_mail_provider_cf.create_mailbox()` 返回的 `meta` 对象

**必需字段**:
- `provider_jwt`: CF Worker 返回的 JWT token (用于邮件读取鉴权)
- `provider_mailbox_id`: CF Worker 内部邮箱 ID (用于删除邮箱)
- `address_id`: CF Worker address_id (备用,部分版本与 provider_mailbox_id 相同)

**可选字段**:
- `provider_name`: "cloudflare_temp_mail"
- `provider_capabilities`: {"delete_mailbox": true, "delete_message": true, ...}

**加密存储**: 
- 本期按 PRD 约束：`temp_mail_meta` 以 JSON 字符串明文存储（JWT 已签名，且本期不引入额外加密与迁移复杂度）
- 后续如需增强安全性，可将 `provider_jwt` 单独加密（例如 `encrypt_data()`），但需要同步更新解析逻辑与兼容旧数据

## 4. 接口设计

### 4.1 `/api/external/pool/claim-random` 扩展

#### 请求

```http
POST /api/external/pool/claim-random
Headers:
  X-API-Key: your_api_key
Content-Type: application/json

Body:
{
  "caller_id": "reg_script_001",      // 必需
  "task_id": "task_001",              // 必需
  "provider": "cloudflare_temp_mail", // 可选,新增
  "project_key": "project_A",         // 可选,现有
  "email_domain": "cf-domain.com"     // 可选,现有
}
```

#### 响应 (成功)

```json
{
  "success": true,
  "code": "OK",
  "message": "success",
  "data": {
    "account_id": 123,
    "email": "abc123@cf-domain.com",
    "claim_token": "clm_xxx",
    "provider": "cloudflare_temp_mail",
    "lease_expires_at": "2026-04-09T10:10:00Z",
    "claimed_at": "2026-04-09T10:00:00Z"
  }
}
```

#### 响应 (失败 - CF Worker 未配置)

```json
{
  "success": false,
  "code": "TEMP_MAIL_PROVIDER_NOT_CONFIGURED",
  "message": "CF Worker base_url 未配置",
  "data": null
}
```

#### 响应 (失败 - CF Worker 配额耗尽)

```json
{
  "success": false,
  "code": "UPSTREAM_SERVER_ERROR",
  "message": "CF Worker 创建邮箱失败 HTTP 500",
  "data": {
    "status_code": 500,
    "body": "..."
  }
}
```

#### 行为规范

1. **provider 参数为空或未传**:
   - 行为不变,从 pool 中随机领取 Outlook/IMAP 邮箱
   - SQL: `WHERE provider IN ('outlook', 'imap') AND pool_status='available'`

2. **provider='cloudflare_temp_mail'**:
   - 优先从 pool 中查找: `WHERE provider='cloudflare_temp_mail' AND pool_status='available'`
   - 如果没有可用,调用 `_create_cf_temp_email_for_pool()`
   - 创建成功后,直接写入 accounts 表,状态为 `claimed`

3. **email_domain 参数**:
   - 如果传入,优先使用指定域名创建 CF 邮箱
   - 如果未传,使用 CF Worker 默认域名

4. **向后兼容**:
   - 不传 provider 参数时,行为完全不变
   - 现有 Outlook/IMAP 邮箱的 claim 流程不受影响

### 4.2 `/api/external/pool/claim-complete` 扩展

#### 请求

```http
POST /api/external/pool/claim-complete
Headers:
  X-API-Key: your_api_key
Content-Type: application/json

Body:
{
  "account_id": 123,
  "claim_token": "clm_xxx",
  "caller_id": "reg_script_001",
  "task_id": "task_001",
  "result": "success",  // ← 决定是否删除远程 CF 邮箱
  "detail": "注册成功"
}
```

#### 响应

```json
{
  "success": true,
  "code": "OK",
  "message": "success",
  "data": {
    "new_status": "used"
  }
}
```

#### 行为规范

1. **更新 pool_status** (现有逻辑):
   - `result='success'` → `pool_status='used'`
   - `result='verification_timeout'` → `pool_status='cooldown'`
   - `result='provider_blocked'` → `pool_status='frozen'`
   - `result='credential_invalid'` → `pool_status='retired'`
   - `result='network_error'` → `pool_status='available'`

2. **删除远程 CF 邮箱** (新增逻辑):
   - 如果 `provider='cloudflare_temp_mail'` **且** `result` 需要删除:
     - `result='success'` → 删除
     - `result='credential_invalid'` → 删除
     - 其他 result → 不删除
   - 调用 `_delete_cf_remote_mailbox(account)`
   - 删除失败 **非阻塞**,记录日志,本地状态仍正常更新

3. **审计日志**:
   - 记录 complete 动作（external_pool controller 层统一 audit）
   - 远程删除为“尽力而为”的非阻塞行为：当前实现仅记录应用日志，不作为外部审计字段的硬要求

### 4.3 邮件读取接口 (无需改动)

#### `/api/external/messages/latest`

```http
GET /api/external/messages/latest?email=abc@cf-domain.com&keyword=验证码

  X-API-Key: your_api_key
```

**行为（以当前代码为准）**:

## 5. 核心流程

### 5.1 动态创建 CF 邮箱流程

```python
# pool.py

def claim_atomic(conn, caller_id, task_id, provider=None, ...):
    """
    邮箱池 claim 原子操作
    """
    # 1. 构建 SQL (根据 provider 过滤)
    sql = "SELECT * FROM accounts WHERE pool_status='available' AND status='active'"
    params = []
    
    if provider:
        sql += " AND provider = ?"
        params.append(provider)
    
    sql += " ORDER BY RANDOM() LIMIT 1"
    
    # 2. 尝试从 pool 中查找可用邮箱
    conn.execute("BEGIN IMMEDIATE")
    account = conn.execute(sql, params).fetchone()
    
    if account:
        # 3a. 如果有可用,直接 claim
        return _claim_existing_account(conn, account, caller_id, task_id)
    
    # 3b. 如果 provider 是 CF 且无可用,动态创建
    if provider == 'cloudflare_temp_mail':
        account = _create_cf_temp_email_for_pool(conn, caller_id, task_id)
        conn.execute("COMMIT")
        return account
    
    # 3c. 其他 provider 无可用时,回滚事务
    conn.execute("ROLLBACK")
    return None


def _create_cf_temp_email_for_pool(conn, caller_id, task_id):
    """
    为 pool 动态创建 CF 临时邮箱
    """
    from outlook_web.services.temp_mail_provider_cf import CloudflareTempMailProvider
    import secrets
    
    # 1. 调用 CF Provider 创建邮箱
    provider = CloudflareTempMailProvider()
    result = provider.create_mailbox()  # 自动生成随机前缀 + 默认域名
    
    if not result['success']:
        raise PoolServiceError(result['error'], result['error_code'])
    
    # 2. 生成 claim token 和时间戳
    token = 'clm_' + secrets.token_urlsafe(9)
    now_str = _utcnow().isoformat() + 'Z'
    lease_expires_at_str = (_utcnow() + timedelta(seconds=600)).isoformat() + 'Z'
    
    # 3. 写入 accounts 表 (claimed 状态)
    conn.execute("""
        INSERT INTO accounts (
            email, provider, pool_status, temp_mail_meta,
            claim_token, claimed_by, claimed_at, lease_expires_at,
            last_claimed_at, created_at, updated_at, status
        ) VALUES (?, ?, 'claimed', ?, ?, ?, ?, ?, ?, ?, ?, 'active')
    """, (
        result['email'],
        'cloudflare_temp_mail',
        json.dumps(result['meta']),  # 存储 JWT 等元数据
        token,
        f"{caller_id}:{task_id}",
        now_str,
        lease_expires_at_str,
        now_str,
        now_str,
        now_str
    ))
    
    account_id = conn.lastrowid
    
    # 4. 记录 claim log
    conn.execute("""
        INSERT INTO account_claim_logs
            (account_id, claim_token, caller_id, task_id, action, result, detail, created_at)
        VALUES (?, ?, ?, ?, 'claim', NULL, 'CF邮箱动态创建', ?)
    """, (account_id, token, caller_id, task_id, now_str))
    
    # 5. 返回完整 account 信息
    return {
        "id": account_id,
        "email": result['email'],
        "provider": 'cloudflare_temp_mail',
        "pool_status": 'claimed',
        "claim_token": token,
        "claimed_at": now_str,
        "lease_expires_at": lease_expires_at_str,
        "temp_mail_meta": json.dumps(result['meta'])
    }
```

### 5.2 删除远程 CF 邮箱流程

```python
# pool.py

def complete_claim(account_id, claim_token, caller_id, task_id, result, detail=None):
    """
    标记领取结果并驱动状态机流转
    """
    conn = create_sqlite_connection()
    try:
        # 1. 获取 account 信息
        account = conn.execute(
            "SELECT * FROM accounts WHERE id = ?", 
            (account_id,)
        ).fetchone()
        
        if not account:
            raise PoolServiceError("账号不存在", "account_not_found")
        
        # 2. 校验 claim_token (现有逻辑)
        if account['claim_token'] != claim_token:
            raise PoolServiceError("claim_token 不匹配", "token_mismatch")
        
        # 3. 更新 pool_status (现有逻辑)
        new_status = RESULT_TO_POOL_STATUS[result]
        conn.execute(
            "UPDATE accounts SET pool_status = ?, updated_at = ? WHERE id = ?",
            (new_status, _utcnow().isoformat() + 'Z', account_id)
        )
        
        # 4. 记录 claim log (现有逻辑)
        conn.execute("""
            INSERT INTO account_claim_logs
                (account_id, claim_token, caller_id, task_id, action, result, detail, created_at)
            VALUES (?, ?, ?, ?, 'complete', ?, ?, ?)
        """, (account_id, claim_token, caller_id, task_id, result, detail, _utcnow().isoformat() + 'Z'))
        
        conn.execute("COMMIT")
        
        # 5. 如果是 CF 邮箱且需要删除,调用 CF Provider 删除 (新增逻辑)
        if account['provider'] == 'cloudflare_temp_mail' and _should_delete_cf_mailbox(result):
            _delete_cf_remote_mailbox(account)
        
        return new_status
        
    finally:
        conn.close()


def _should_delete_cf_mailbox(result: str) -> bool:
    """
    判断是否需要删除远程 CF 邮箱
    """
    return result in ('success', 'credential_invalid')


def _delete_cf_remote_mailbox(account):
    """
    删除远程 CF 邮箱 (非阻塞)
    """
    from outlook_web.services.temp_mail_provider_cf import CloudflareTempMailProvider
    import logging
    
    logger = logging.getLogger(__name__)
    
    try:
        # 1. 解析 temp_mail_meta
        meta_raw = account.get('temp_mail_meta') or '{}'
        meta = json.loads(meta_raw)
        
        # 2. 构建 mailbox 对象
        mailbox = {
            'email': account['email'],
            'meta': meta
        }
        
        # 3. 调用 CF Provider 删除
        provider = CloudflareTempMailProvider()
        success = provider.delete_mailbox(mailbox)
        
        if success:
            logger.info(
                f"[pool] 已删除 CF 远程邮箱: {account['email']} "
                f"(account_id={account['id']})"
            )
        else:
            logger.warning(
                f"[pool] 删除 CF 远程邮箱失败: {account['email']} "
                f"(account_id={account['id']}), 返回 False"
            )
    
    except Exception as e:
        logger.warning(
            f"[pool] 删除 CF 远程邮箱异常: {account['email']} "
            f"(account_id={account['id']}), error={e}"
        )
        # 非阻塞,继续执行,本地状态已更新
```

## 8. 错误处理

### 8.1 错误码定义

| 错误码 | HTTP状态 | 说明 | 场景 |
|--------|---------|------|------|
| `TEMP_MAIL_PROVIDER_NOT_CONFIGURED` | 400 | CF Worker 未配置 | claim 时检测到配置缺失 |
| `UPSTREAM_TIMEOUT` | 500 | CF Worker 请求超时 | create_mailbox 超时 (30s) |
| `UPSTREAM_SERVER_ERROR` | 500 | CF Worker 服务错误 | CF Worker 返回 5xx |
| `UPSTREAM_BAD_PAYLOAD` | 500 | CF Worker 返回格式错误 | 返回非 JSON 或缺少字段 |
| `UNAUTHORIZED` | 401 | CF Worker 鉴权失败 | admin_key 错误 |
| `no_available_account` | 200 | 池中无可用邮箱 | 现有错误码,兼容 |

### 8.2 日志记录

**关键操作日志**:
1. CF 邮箱创建: `[pool] 为 pool 动态创建 CF 邮箱: xxx@domain.com`
2. CF 邮箱删除成功: `[pool] 已删除 CF 远程邮箱: xxx@domain.com`
3. CF 邮箱删除失败: `[pool] 删除 CF 远程邮箱失败: xxx@domain.com, error=...`

**日志级别**:
- 创建成功: INFO
- 删除成功: INFO
- 删除失败: WARNING (非阻塞,本地状态已更新)
- CF Worker 配置缺失: WARNING
- CF Worker 请求异常: ERROR

## 9. 性能与可靠性

### 9.1 性能指标

| 指标 | 目标 | 实现策略 |
|------|------|----------|
| claim 响应时间 | < 3s | CF Worker 创建邮箱耗时 1-2s,可接受 |
| complete 响应时间 | < 2s | 删除操作非阻塞,主线程立即返回 |
| 并发创建 | 支持 10+ QPS | CF Worker 内部处理并发,我们无需限制 |

### 9.2 可靠性保证

**创建失败处理**:
1. CF Worker 返回错误 → 不写入 accounts 表,回滚事务
2. 返回明确错误码给调用方
3. 记录 ERROR 日志

**删除失败处理**:
1. **非阻塞**: 本地 pool_status 已更新,删除失败不影响业务流程
2. 记录 WARNING 日志,包含 account_id 和 email
3. 定期人工检查日志,手动清理失败邮箱 (P2)

**超时处理**:
- CF Worker 请求超时 30s (已在 `temp_mail_provider_cf.py` 定义)
- 超时后返回 `UPSTREAM_TIMEOUT` 错误码

## 11. 迁移计划

### 11.1 Schema 迁移

**文件**: `outlook_web/db.py`

**迁移代码** (v19):
```python
# db.py init_db()

# Schema v19: CF临时邮箱接入邮箱池
if current_version < 19:
    cursor.execute("PRAGMA table_info(accounts)")
    columns = [col[1] for col in cursor.fetchall()]
    
    if "temp_mail_meta" not in columns:
        cursor.execute("ALTER TABLE accounts ADD COLUMN temp_mail_meta TEXT")
        logger.info("[迁移] v19: accounts 表新增 temp_mail_meta 字段")
    
    # 更新 schema 版本
    cursor.execute(
        "INSERT OR REPLACE INTO settings (key, value) VALUES (?, ?)",
        (DB_SCHEMA_VERSION_KEY, "19")
    )
    logger.info("[迁移] Schema 版本更新至 v19")
```

### 11.2 数据迁移

**不迁移 `temp_emails` 表历史数据**:
- 原因: 数据量小,历史 CF 邮箱可保持只读
- 策略: 新创建的 CF 邮箱直接写入 `accounts` 表
- 兼容: `temp_emails` 表保留,前端可继续查看历史数据
