## 关键技术要点

### 1. 注册表机制

```python
# temp_mail_provider_base.py
_REGISTRY: dict[str, type[TempMailProviderBase]] = {}

def register_provider(cls):
    name = getattr(cls, "provider_name", "")
    if isinstance(name, str) and name.strip():
        _REGISTRY[name.strip()] = cls
    return cls
```

### 2. 插件加载

使用 `importlib.util.spec_from_file_location()`，模块名加 `_plugin_` 前缀避免命名冲突。热刷新时先清空第三方注册 + 清理 `sys.modules` 再重新扫描。

### 3. 内置 Provider 保护

`_BUILTIN_PROVIDERS = {"cloudflare_temp_mail", "custom_domain_temp_mail", "legacy_bridge"}` — 热刷新只清空不在此白名单中的条目。

### 4. 配置隔离

插件配置 key 格式：`plugin.{provider_name}.{field_key}`，通过 `settings_repo.get_setting()` 读写。

### 5. 卸载安全

卸载前检查 `temp_emails` 表中 `source = provider_name AND status = 'active'`。有任务邮箱（`mailbox_type = 'task'`）时阻止卸载。

### 6. 错误处理

使用 `PluginManagerError(code, message, status=xxx, data=xxx)` 自定义异常，控制器捕获后通过 `build_error_payload()` 转为统一错误响应。
